# 폼 레이아웃 간격 최적화

**업데이트 날짜**: 2025-11-17
**파일**: index.html

---

## 🎯 문제점

상품권 금액 입력과 고객정보 입력 사이의 간격이 너무 넓어서 레이아웃이 산만하고 비효율적이었습니다.

### Before (문제 상황)
```
01. 상품권 종류 선택
[30px 여백]
[상품권 리스트]

02. 핀번호 및 금액
[20px 여백]
[입력 필드]
[20px 여백]
[입력 필드]
[30px 여백]
[계산 결과]
[30px 여백]

03. 고객 정보
[20px 여백]
[입력 필드]
```

### After (개선 결과)
```
01. 상품권 종류 선택
[12px 여백]
[상품권 리스트]

02. 핀번호 및 금액
[16px 여백]
[입력 필드]
[16px 여백]
[입력 필드]
[20px 여백]
[계산 결과]
[24px 여백]

03. 고객 정보
[16px 여백]
[입력 필드]
```

## ✅ 수정 내용

### 1. 폼 섹션 타이틀 간격 조정
```css
/* Before */
.form-section-title {
    margin-top: 30px;
}

/* After */
.form-section-title {
    margin-top: 24px;
    margin-bottom: 12px;
}

.form-section-title:first-of-type {
    margin-top: 0;  /* 첫 번째는 위 여백 없음 */
}
```

**변경 사항**:
- 위 여백: `30px` → `24px` (6px 감소)
- 아래 여백: 명시적으로 `12px` 추가
- 첫 번째 섹션: 위 여백 `0`으로 설정

---

### 2. 폼 그룹 간격 조정
```css
/* Before */
.form-group {
    margin-bottom: 20px;
}

/* After */
.form-group {
    margin-bottom: 16px;
}
```

**변경 사항**:
- 아래 여백: `20px` → `16px` (4px 감소)

---

### 3. 계산 결과 표시 영역 여백 조정
```css
/* Before */
.calculation-display {
    margin: 30px 0;
}

/* After */
.calculation-display {
    margin: 20px 0 24px;
}
```

**변경 사항**:
- 위 여백: `30px` → `20px` (10px 감소)
- 아래 여백: `30px` → `24px` (6px 감소)

---

### 섹션별 절약 공간
```

폼 그룹 (5개): 4px × 5 = 20px
계산 표시: 16px (위아래 합)
기타: 8px

총 절약: ~62px
```

- **시각적 리듬**: 일관된 간격으로 깔끔함
- **집중도**: 산만함 감소

- **L**: 16px (폼 그룹, 전체 여백)
- **XL**: 20-24px (섹션 구분, 계산 결과)
